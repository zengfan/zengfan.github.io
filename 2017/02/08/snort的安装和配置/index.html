snort的安装和配置

发表于 2017-02-08 | 分类于 入侵检测 | 本文总阅读量 次

snort的安装和配置挺麻烦，这里记下一年前配环境的过程。

一、snort安装

安装snort之前需要安装库：flex、bison、libpcap、DAQ、libpcre-dev、libdumbnet-dev 和zlib

apt-get install flex
aptitude install libpcap-dev
apt-get install bison

然后源码安装DAQ

aptitude install libpcre3-dev
aptitude install libdumbnet-dev
aptitude install zlib1g-dev

然后源码安装snort

创建安装目录并在脚本中设置prefix参数。

#mkdir /usr/local/snort

#./configure –prefix=/usr/local/snort/ –enable-sourcefire

#make

#make install
最后，从/usr/local/snort/bin中运行snort。现在它对eth0的所有流量都处在promisc模式（包转储模式）。

snort running
如下图所示snort转储流量。

二、snort配置

创建需要的文件和目录

#mkdir /etc/snort

#mkdir /var/log/snort

#mkdir /usr/local/lib/snort_dynamicrules

#mkdir /etc/snort/rules

#touch /etc/snort/rules/white_list.rules /etc/snort/rules/black_list.rules

#cp /root/snort-2.9.7.0/etc/gen-msg.map threshold.conf classification.config reference.config unicode.map snort.conf /etc/snort/

ln -s /usr/local/snort/lib/snort_dynamicpreprocessor /usr/local/lib/snort_dynamicpreprocessor
ln -s /usr/local/snort/lib/snort_dynamicengine /usr/local/lib/snort_dynamicengine
ln -s /usr/local/snort/lib/snort_dynamicrules /usr/local/lib/snort_dynamicrules

ln -s /usr/local/snort/bin/snort /usr/sbin/

编辑配置文件

#vi /etc/snort/snort.conf
修改路径变量
var RULE_PATH /etc/snort/rules
var SO_RULE_PATH /etc/snort/so_rules
var PREPROC_RULE_PATH /etc/snort/preproc_rules
var WHITE_LIST_PATH /etc/snort/rules
var BLACK_LIST_PATH /etc/snort/rules
设置log目录
config logdir：/var/log/snort
配置输出插件
output unified2：filename snort.log，limit 128

配置默认规则

#tar zxvf snortrules-snapshot-2970.tar.gz -C /etc/snort/

#cp /etc/snort/etc/sid-msg.map /etc/snort/

测试snort

#snort -T -i eth0 -c /etc/snort/snort.conf

参数解释：
-T 指定启动模式：测试
-i 指定网络接口
-c 指定配置文件

如果出现“success”的字样说明配置好了
按ctrl+c终止测试

三、安装barnyard2

先装mysql

apt-get install libmysqlclient15-dev

#cd /root/barnyard2-1.9
autoreconf -fvi -I ./m4 ./autogen.sh 就可以了。。。。

#./configure –with-mysql –with-mysql-libraries=/usr/lib64/mysql/

#make && make install

./configure –with-mysql support for mysql
./configure –with-mysql-libraries=DIR mysql library directory

If you are using a 32-bit system enter the following command (Refer to the instructions above on how to determine if you are using a 32-bit or 64-bit system):
./configure –with-mysql –with-mysql-libraries=/usr/lib/i386-linux-gnu/
If you are using a 64-bit system enter the following command (Refer to the instructions above on how to determine if you are using a 32-bit or 64-bit system):
./configure –with-mysql –with-mysql-libraries=/usr/lib/x86_64-linux-gnu/

在arm板子中，find -name libmysqlclient.so

/usr/lib/arm-linux-gnuebihf/libmysqlclient.so

四、配置barnyard2

创建需要的文件和目录

#mkdir /var/log/barnyard2

#touch /var/log/snort/barnyard2.waldo

#cp /root/barnyard2-1.9/etc/barnyard2.conf /etc/snort
修改配置文件

#vi /etc/snort/barnyard2.conf
config logdir:/var/log/barnyard2
config hostname:localhost
config interface:eth0
config waldo_file:/var/log/snort/barnyard.waldo
output database: log, mysql, user=snort password=123456 dbname=snort host=localhost

ln -s /usr/local/bin/barnyard2 /usr/bin/

测试barnyard2

#barnyard2 -c /etc/snort/barnyard2.conf -d /var/log/snort -f snort.log -w /var/log/snort/barnyard2.waldo -u root -g root

snort -q -c /etc/snort/snort.conf -i eth0 -l /var/log/snort -u root -g root

参数解释：
-c 指定配置文件
-d 指定log目录
-f 指定log文件
-w 指定waldo文件

如果出现“Waiting for new spool file”字样则表示barnyard2配置成功
按ctrl+c终止测试

五、mysql

cp barnyard2-2-1.13/schemas/create_mysql /usr/src
create database snort;
use snort;

source /usr/src/create_mysql;
执行以下命令，配置mysql远程访问：
GRANT ALL PRIVILEGES ON . TO ‘root’@’%’ IDENTIFIED BY ‘snort’ WITH GRANT OPTION;
FLUSH PRIVILEGES;
重启msyql服务即可远程访问
hadoop@master:~$ sudo service mysql restart
mysql stop/waiting
mysql start/running, process 32359

关闭防火墙：sudo service ufw stop

六、规则

注释掉除去local.rules的所有规则。
注释sid-msg.map中的msg信息，并添加以下信息，不然会影响barnyard2启动效率
499 || ICMP Large ICMP packet
100888 || web access alert
100333 || possible scan port 23
104444 || abnormal network connetction
100555 || possible TCP DoS attack
100666 || possible TCP DDoS attack
注释gen-msg.map中的除以下3条的所有msg信息，不然会影响barnyard2启动效率

1 || 1 || snort general alert
2 || 1 || tag: Tagged Packet
3 || 1 || snort dynamic alert
在local.rules中添加规则：
alert tcp 192.168.1.100 any -> $HOME_NET any
(msg:”abnormal network connection”; classtype:bad-unknown; sid:104444; rev:1;)

alert tcp any any -> $HOME_NET 24 (msg:”test:port scan 230”; sid:100333; classtype:bad-unknown; rev:1;)
alert tcp any any -> 202.108.22.5 any (msg:”web access”; sid:100088; classtype:bad-unknown; rev:1;)
alert icmp $EXTERNAL_NET any -> $HOME_NET any (msg:”ICMP Large ICMP Packet”; dsize:>800;
reference:arachnids,246; classtype:bad-unknown; sid:499; rev:4;)

alert tcp $EXTERNAL_NET any -> 192.168.1.72 80 (flags:S; msg:”possible TCP DoS”; flow: stateless;
threshold:type both, track by_src, count 70, seconds 10; classtype:bad-unknown; sid:100555; rev:1;)

alert tcp $EXTERNAL_NET any -> $HOME_NET 80 (flags:S; msg:”possible TCP DDoS”; flow: stateless;
threshold:type both, track by_dst, count 70, seconds 10; classtype:bad-unknown; sid:100666; rev:1;)

七、启动

test.sh

rm -rf /var/log/snort/snort.log.*

snort -q -c /etc/snort/snort.conf -i eth0 -l /var/log/snort -u root -g root &

barnyard2 -c /etc/snort/barnyard2.conf -d /var/log/snort -f snort.log -w /var/log/snort/barnyard2.waldo -u root -g root

八、测试

异常访问：hping3 -I eth0 -a 192.168.1.100 -S 192.168.1.72

Dos：hping3 -I eth0 -S 192.168.1.72 -p 80 -i u1000

DDoS: hping3 -I eth0 –rand-source -S 192.168.1.72 -p 80 -i u1000

ICMP flood: ping 192.168.1.72 -l 801 (需要用windows执行此命令)

ping 192.168.1.72 -c 4 -s 888 (linux)

端口扫描：nmap 192.168.1.72

demo2测试:

异常访问（外部访问板子）：hping3 -I eth0 -a 192.168.1.100 -S 192.168.1.111

ICMP flood: ping 192.168.1.111 -c 2 -s 888

异常访问（板子访问外部，在被攻击的板子上执行）：wget 202.108.22.5
